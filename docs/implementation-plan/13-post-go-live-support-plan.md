# 13. Post-Go-Live Support Plan

A structured support schedule to ensure application stability and user alignment.

### 1. Phase 1: Client Sandbox Testing Support (Up to 2 Months Free)
*   **Timeline:** Begins immediately following implementation completion.
*   **Scope:** Providing guidance and resolving queries during the client's team-led sandbox validation.
*   **Response Times:**
    *   *Staging Blocker (e.g. UAT workflow broken):* Responded to within 4 hours.
    *   *Staging Minor (e.g. Print template layout adjustment):* Responded to within 24 hours.

### 2. Phase 2: Production Maintenance & Server Support (2 Months Free)
*   **Timeline:** Commences immediately upon system Go-Live in production.
*   **Scope:** Dedicated to infrastructure stability, database safety, and bug correction.
    *   *Server Uptime:* Continuous monitoring of CPU, RAM, and disk utilization.
    *   *Backup Administration:* Daily automated off-site database backups.
    *   *Minor Bug Fixes:* Immediate patch deployments for errors occurring in production POS, accounting posting, or payroll calculations.
*   **Exclusions:** Adding new features or re-architecting workflows after go-live is excluded from free support and will require a change request.
